请求参数

参数名 类型 是否必须 描述

POST /user/jobseeker-update-info 更新求职者个人信息

更新当前登录求职者的个人资料信息。

请求头

参数名 类型 是否必须 描述
Authorization String 必须 Bearer 令牌

请求参数

参数名 类型 是否必须 描述
mobile String 可选 手机号码
email String 可选 电子邮箱地址
avatar String 可选 头像图片URL
realName String 可选 真实姓名
gender String 可选 性别（male：男，female：女）
birthday String 可选 出生日期（YYYY-MM-DD格式）
expectPosition String 可选 期望职位
workYears Integer 可选 工作年限
skill String 可选 专业技能

resumeUrl String 可选 简历URL（PDF或DOCX格式）

    "createTime": "2023-04-01T10:30:00",

    "gender": "male",
    "birthday": "1990-01-01",

GET /user/verify-token 验证令牌有效性

验证用户令牌的有效性，并返回用户角色信息，可用于前端权限控制和验证登录状态。

请求头

参数名 类型 是否必须 描述
Authorization String 必须 Bearer 令牌

响应示例 - 有效令牌

JSON 
{
  "code": 0,
  "data": {
    "valid": true,
    "userId": 3,
    "username": "zhangsan",
    "role": "job_seeker"
  },
  "message": "success"
}

响应示例 - 无效令牌

JSON 
{  "code": 0,
  "data": {
    "valid": true,
    "userId": 1,
    "username": "user123",
    "role": "job_seeker"
  },
  "message": "success"
}

说明

  • 令牌无效或过期时，实际上可能会由JWT过滤器直接拦截并返回401错误
  • 当用户在应用中操作时，可定期调用此接口检验令牌是否仍然有效
  • 可根据返回的用户角色信息进行前端页面权限控制

企业模块

企业注册、管理和信息检索相关接口

POST /company/register 注册新企业及管理员

注册新企业并创建企业管理员账号。成功返回"注册成功"提示。

请求参数

参数名 类型 是否必须 描述
name String 必须 企业名称（最多100个字符）
licenseNumber String 必须 营业执照号（18位）
address String 可选 企业地址（最多200个字符）
legalPerson String 必须 法定代表人姓名（最多50个字符）
username String 必须 企业管理员用户名（4-50个字符）
password String 必须 企业管理员密码（6-20个字符）
email String 必须 企业管理员电子邮箱地址
position String 可选 企业管理员职位（最多50个字符）

请求示例

JSON 
{
  "name": "北京智慧科技有限公司",
  "licenseNumber": "91110105MA00B4XC2X",
  "address": "北京市海淀区科技园区123号",
  "legalPerson": "李明",
  "username": "admin1",
  "password": "admin123",
  "email": "admin@zhihuikeji.com",
  "position": "总经理"
}

响应示例

JSON 
{
  "code": 0,
  "data": "注册成功",
  "message": "success"
}

POST /company/add-project-manager 添加项目经理

向企业添加项目经理账号。注意：只有企业管理员才能添加项目经理。

请求头

参数名 类型 是否必须 描述
Authorization String 必须 Bearer 令牌（企业管理员令牌）

请求参数

参数名 类型 是否必须 描述
companyId Integer 必须 企业ID
username String 必须 项目经理用户名（4-50个字符）
password String 必须 项目经理密码（6-20个字符）
email String 必须 项目经理电子邮箱地址
position String 可选 项目经理职位（最多50个字符）

请求示例

JSON 
{
  "companyId": 1,
  "username": "manager1",
  "password": "manager123",
  "email": "manager1@zhihuikeji.com",
  "position": "技术项目经理"
}

响应示例

JSON 
{
  "code": 0,
  "data": "添加成功",
  "message": "success"
}

错误响应示例

JSON 
{
  "code": 40300,
  "data": null,
  "message": "只有企业管理员才能添加项目经理"
}
